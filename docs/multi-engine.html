Multi-Engine Support

The framework separates behavioral specs (what the game does) from engine bindings (how a specific engine implements it), enabling the same game design to target multiple engines.

Architecture

  • Behavioral specs (specs/mechanics/, specs/dynamics/, etc.) are engine-agnostic
  • Binding specs (specs/bindings/) map behavioral contracts to engine-specific APIs
  • Equivalence table (specs/bindings/equivalence.md) provides a quick cross-engine reference

When implementing for a specific engine, read both the mechanic spec (what to build) and its binding spec (how to build it in your engine).
