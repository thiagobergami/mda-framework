Game Concept Schema

The starting point of the entire framework. Before any MDA breakdown happens, you need a clear picture of: what is this game, who is it for, and what experience does it deliver?

File pattern: specs/concept/{game-name}.concept.md
ID format: GAME-{NNN}
Scaffold: npx mda new concept "Game Title"

Template

Frontmatter

---
id: GAME-{NNN}
name: {Game title or working title}
version: {Spec version}
---

Required Sections

Vision

2-4 sentences capturing the essence of the game. The elevator pitch. A reader should immediately understand what kind of experience this is.

Aesthetic Profile

Rank the 8 aesthetic categories by importance for this game. This profile drives every design decision downstream.

PriorityAestheticRole in this game
Primary{category}How this aesthetic manifests
Secondary{category}How it supports the primary
Tertiary{category}Present but not a driver
Absent{category}Explicitly NOT pursued

Include a Conflicts and resolutions subsection if any aesthetics tension each other.

Core Loop

The repeating cycle of player activity. Describe as: action → feedback → decision → action.

{Action} → {System Response} → {Player Decision} → {repeat}

- Frequency: How often this cycle repeats
- Serves aesthetic: Which primary/secondary aesthetic

Target Audience

Age range, player archetype, experience level, session length, social context.

Platform Constraints

Runtime, language, target devices, performance budget, input methods, network model.

Feature Map

High-level list of features, each connecting to an aesthetic. This becomes the spec backlog.

FeaturePrimary AestheticPriorityStatus
{name}{aesthetic}Must-have / Should-have / Nice-to-haveNot started / In progress / Specced / Implemented

Boundaries

What this game is NOT. Explicit exclusions prevent scope creep and help AI make judgment calls.

Reference Games

Games that share aspects of the target experience. Be specific about what to learn from — not "it's like Minecraft" but "Minecraft's Discovery loop."

Success Criteria

High-level metrics that map to the aesthetic profile. These become the source of truth for aesthetic spec observable proxies.

Rules

  1. Write the concept FIRST — Before any AES, DYN, MEC, TUN, or AST spec
  2. Rank aesthetics explicitly — "Primary" means trade-offs favor this. "Absent" means actively avoid it
  3. One core loop — If you have two equally important loops, you have two games
  4. Boundaries prevent scope creep — Easier for AI to implement "a Discovery game that is NOT a combat game" than just "a Discovery game"
  5. Reference games are specific — "Like Zelda" is useless. "Zelda BotW's emergent physics interactions" is actionable
  6. Feature map is the spec backlog — Each row generates a full AES→DYN→MEC→TUN chain

Quality Gate: Concept Readiness

npx mda gate concept
CheckWhat it verifies
Vision clarityUses aesthetic vocabulary, not vague words like "fun"
Aesthetic commitmentPrimary + Absent tiers defined with enough Absent entries
Core loop coherenceLoop serves the Primary aesthetic
Boundary definitionAt least 3 boundaries with aesthetic rationale
Feature traceabilityAll features connect to aesthetics
Scope realismNot too many Must-have features
