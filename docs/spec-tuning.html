Tuning Spec Schema

Tuning specs define the iterative refinement parameters for a feature. They answer: "What values can we adjust, within what ranges, to improve the experience?"

File pattern: specs/tuning/{feature}.tune.md
ID format: TUN-{NNN}
Scaffold: npx mda new tuning "Feature Pacing"

From the MDA paper

"By iteratively refining the value of penalties, rate of taxation or thresholds for rewards and punishments, we can refine the gameplay until it is balanced."

Template

Frontmatter

---
id: TUN-{NNN}
name: {Descriptive name}
traces_to_mechanics: [MEC-NNN]
traces_to_dynamics: [DYN-NNN]
traces_to_aesthetics: [AES-NNN]
---

Tuning specs are the only layer that traces to ALL three other layers, because tuning affects the entire M→D→A chain.

Required Sections

Tuning Goal

What experience outcome this tuning achieves. Reference the aesthetic spec and define what "balanced" means.

Parameters

Every [TUNABLE] value from related mechanic specs.

### peekThreshold
- Mechanic: MEC-005, Rule 2
- Current value: 0.15
- Range: [0.05, 0.40]
- Step: 0.05
- Affects dynamic: Light Reveal Cycle peek phase duration
- Affects aesthetic: Creature Wonder — anticipation before reveal
- Sensitivity: High

Target Metrics

Measurable outcomes corresponding to aesthetic observable proxies.

MetricTargetToleranceSource
reveal_pause1.0s±0.3sAES-003
discovery_rate25s±10sAES-002

Tuning Constraints

Rules limiting how parameters relate to each other.

Known Trade-offs

Adjustments that improve one aesthetic at the expense of another.

- peekThreshold vs revealSpeed: Lower peek threshold increases anticipation (Sensation)
  but slows down discovery rate (Discovery). Primary aesthetic (Discovery) wins.

Iteration Log

Record of tuning changes and observed effects. Builds institutional knowledge.

### Iteration 1 — 2026-04-14
- Changed: peekThreshold from 0.30 to 0.15
- Reason: INV-2 failures — light jumps skipping peek phase
- Observed: reveal_pause improved from 0.3s to 1.2s
- Decision: Keep

Rules

  1. Every tunable parameter is documented — Hidden magic numbers are the enemy of iteration
  2. Ranges prevent extremes — AI must never set values outside the range
  3. Sensitivity guides priority — High-sensitivity parameters need small, careful adjustments
  4. Trade-offs are explicit — Name what breaks when you "fix" something
  5. Log iterations — Past attempts (including failures) inform future adjustments
  6. Trace in all directions — Tuning affects the entire M→D→A chain
