MDA Spec Framework

A spec-driven development framework for AI-assisted game development, built on the MDA framework (Mechanics, Dynamics, Aesthetics) by Hunicke, LeBlanc & Zubek.

Get Started Learn the Concepts

The Problem

AI coding assistants can write game code, but they lack design intent. They implement features "in vacuo" — isolated from their effect on player experience. A damage system gets built without knowing it exists to create tension. A hint system gets tuned without knowing it serves discovery.

The Solution

Structure game design as layered, traceable specifications that give AI three things it normally lacks:

🎯

Intent

Aesthetic specs explain why a feature exists — which of the 8 player experience categories it serves.

📚

Constraints

Dynamic specs define behavioral boundaries and feedback systems that must emerge from the mechanics.

✅

Testability

Mechanic specs provide acceptance criteria, and runtime logs validate invariants and aesthetic proxies automatically.

How It Works

Every game feature is specified across the MDA causal chain before code is written:

Game Concept          "A cozy hide-and-seek game for young children"
      |
      v
Aesthetics (WHY)      "The player should feel the delight of discovery"
      |                 Primary: Discovery | Secondary: Fantasy, Sensation
      v
Dynamics (WHAT)       "A negative feedback loop ensures the player always finds
      |                 the baby within 50 seconds via escalating hints"
      v
Mechanics (HOW)       "Rule 4: Hint system fires at [15s, 25s, 35s, 45s] with
      |                 escalating cues. discoveryRange = 8 studs."
      v
Tuning (BALANCE)      "hintTier1Time: 15s [10-25], sensitivity: High
      |                 discoveryRange: 8 studs [5-15], sensitivity: High"
      v
Assets (CONTENT)      "Baby character: 5k tri, R15 rig, 8 emotion blendshapes
                        Status: placeholder | Emotional intent: warmth"

AI reads top-down (M→D→A) to understand intent when implementing. AI reads bottom-up (A→D→M) to trace problems when debugging.

Key Features

🛠

CLI Tooling

Validate specs, run quality gates, and scaffold new specs from the command line with npx mda.

📊

Structured Logging

MDALogger tags every runtime event with its MDA layer and spec ID for traceable debugging.

🎮

Multi-Engine

Engine-agnostic behavioral specs with binding layers for Roblox, Unity, and Unreal Engine.

🔗

Full Traceability

Bidirectional links between all spec layers. Every mechanic traces to a dynamic and an aesthetic.

🚪

Quality Gates

Automated checks ensure specs are complete before proceeding to the next layer.

🤖

AI-Optimized

Designed for AI assistants to read and implement with full design context via CLAUDE.md.

Based On

MDA: A Formal Approach to Game Design and Game Research by Robin Hunicke, Marc LeBlanc, Robert Zubek (GDC 2001-2004).
