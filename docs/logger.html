MDA Logger

Structured runtime logging for Roblox (Luau). Tags every runtime event with its MDA layer and spec ID, enabling traceable debugging from Aesthetics down to Mechanics.

Module: src/shared/MDALogger.luau
Spec: MEC-003

Log Format

[MM:SS.mmm] [LAYER:SPEC_ID] [LEVEL] [EVENT_TYPE] key=value key=value ...

Example output:

[00:15.234] [M:MEC-001] [INFO]  [HINT_FIRED]    cid=3 tier=1 elapsed=15.2s
[00:25.891] [M:MEC-001] [INFO]  [DISCOVERY]     cid=7 spot=kitchen time=25.8s
[00:25.891] [M:MEC-002] [INFO]  [EXPRESSION]    cid=7 emotion=Giggle anim=giggle_02
[00:25.892] [D:DYN-001] [PASS]  [INVARIANT]     cid=7 inv=INV-1 detail="25.8s < 50s"
[03:12.000] [A:AES-001] [PASS]  [PROXY_CHECK]   proxy=discovery_rate value=24.1 target=25

Log Levels

LevelPurposeWhen to use
TRACEPer-frame data, very verbosePosition updates, continuous state (disabled by default)
DEBUGInternal state changesVariable assignments, branch decisions
INFOGame eventsDiscoveries, hints, state transitions
WARNDegraded behaviorNear-miss invariants, metrics drifting
ERRORBroken contractInvariant violations, impossible states
PASSValidated checkInvariant or proxy confirmed
FAILViolated checkInvariant or proxy broken

Core API

Setup

local Log = require(path.to.MDALogger)

Logging Events

-- Convenience methods at each level
Log.trace("M", "MEC-001", "POSITION_UPDATE", { x = 10, y = 5, z = 20 })
Log.debug("M", "MEC-001", "STATE_CHANGE", { from = "idle", to = "searching" })
Log.info("M", "MEC-001", "DISCOVERY", { spot = "kitchen", time = 25.8 })
Log.warn("D", "DYN-001", "METRIC_DRIFT", { metric = "reveal_rate", value = 45 })
Log.err("M", "MEC-001", "IMPOSSIBLE_STATE", { state = "negative_health" })

-- Full signature
Log.log(layer, specId, level, eventType, data, correlationId)

Correlation IDs

Events that happen in the same logical moment share a correlation ID. This lets you trace full causal chains across specs.

local cid = Log.correlate()
Log.info("M", "MEC-001", "DISCOVERY", { spot = "kitchen", time = 25.8 }, cid)
Log.info("M", "MEC-002", "EXPRESSION", { emotion = "Giggle" }, cid)
Log.info("D", "DYN-001", "FEEDBACK", { loop = "DiscoveryAssistance" }, cid)

Invariant Checking

Validate dynamic invariants at runtime. Failures ALWAYS log at ERROR level.

Log.checkInvariant(
  "DYN-001",       -- spec ID
  "INV-1",         -- invariant ID
  time < 50,       -- boolean condition
  "discovery in " .. time .. "s < 50s cap"  -- detail string
)

Proxy Checking

Validate aesthetic observable proxies. Logs PASS, WARN (drifting), or FAIL.

Log.checkProxy(
  "AES-001",         -- spec ID
  "discovery_rate",  -- proxy name
  24.1,              -- measured value
  25,                -- target value
  10                 -- tolerance
)

Metric Tracking

Track metrics over time for session summary computation.

Log.trackMetric("AES-001", "discovery_rate", measuredTime, 25, 10)

Session Summary

Emits aggregate metrics and auto-checks all tracked proxies.

Log.summary()

Output includes median, min, max for each metric plus PASS/FAIL for each proxy.

Configuration

-- Set minimum log level
Log.setLevel(Log.Level.DEBUG)    -- Show DEBUG and above
Log.setLevel(Log.Level.TRACE)    -- Show everything
Log.setLevel(Log.Level.NONE)     -- Disable all output

-- Enable/disable
Log.enable()
Log.disable()

-- Control buffer size (ring buffer, default 2000)
Log.setBufferSize(5000)

-- Control print output
Log.setPrintToOutput(false)  -- Buffer only, no console output

Buffer & Filtering

-- Get all buffered entries in chronological order
local entries = Log.getBuffer()

-- Filter by substring
local failures = Log.getBufferFiltered("[FAIL]")
local specEntries = Log.getBufferFiltered("[M:MEC-001]")
local chain = Log.getBufferFiltered("cid=7")

-- Get invariant pass/fail counts
local results = Log.getInvariantResults()

-- Reset session (clears buffer, metrics, invariants)
Log.reset()

Debugging Workflow (A → D → M)

Use MDA logs to trace from experience to code:

  1. Filter for [FAIL] and [WARN] — find what's broken
  2. Check [A:AES-*] [PROXY_CHECK] — which aesthetic proxy is out of range?
  3. Check [D:DYN-*] [INVARIANT] — which dynamic invariant failed?
  4. Filter by correlation ID (cid=N) — trace the full causal chain
  5. Read [M:MEC-*] in that chain — which mechanic event preceded the failure?
  6. Cross-reference the mechanic spec — which rule or parameter is off?
