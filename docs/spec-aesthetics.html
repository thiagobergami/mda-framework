Aesthetic Spec Schema

Aesthetic specs define the player experience goals for a feature. They answer: "What should the player FEEL when interacting with this part of the game?"

File pattern: specs/aesthetics/{feature}.aes.md
ID format: AES-{NNN}
Scaffold: npx mda new aesthetic "Feature Name"

From the MDA paper

"In describing the aesthetics of a game, we want to move away from words like 'fun' and 'gameplay' towards a more directed vocabulary."

Template

Frontmatter

---
id: AES-{NNN}
name: {Descriptive name}
primary_aesthetic: {One of the 8 categories}
secondary_aesthetics: [{List of supporting categories}]
target_audience: {Who this experience is for}
---

Required Sections

Experience Goal

2-3 sentences describing the desired player experience using aesthetic vocabulary. Never use "fun" or "gameplay."

Aesthetic Profile

Primary aesthetic with its "Game as..." frame, plus secondary aesthetics with explanations of how they support or amplify the primary.

Observable Proxies

Measurable player behaviors that indicate the aesthetic is working. Each proxy has a name, what to measure, and a target range or threshold.

- reveal_pause_duration: Average time players pause at reveals — Target: >1.0s
- discovery_rate: Median time between discoveries — Target: 25s ±10

Anti-Patterns

Behaviors or states that indicate the aesthetic is FAILING. Each anti-pattern must have a detection signal.

Audience Context

Who is the target player? The same mechanics produce different aesthetics for different audiences.

Rules

  1. Use the 8 categories — Every spec must classify using the glossary vocabulary
  2. One primary — Exactly one dominant aesthetic. If you can't choose, the feature is too broad; split it
  3. Observable proxies are required — If you can't define how to observe the aesthetic, it's too vague to implement
  4. Anti-patterns prevent drift — They catch when a mechanic change breaks the experience
  5. Audience matters — The same system produces different aesthetics for different players
  6. No implementation details — Aesthetics describe WHAT the player feels, never HOW the system achieves it

Quality Gate: Aesthetic Readiness

npx mda gate aesthetic
CheckWhat it verifies
Measurable proxiesAll proxies have numeric targets, not subjective phrases
Anti-patterns definedEach anti-pattern has a detection signal
Precise vocabularyNo "fun", "engaging", or other banned vague words
Primary classifiedExactly one primary aesthetic from the 8 categories
