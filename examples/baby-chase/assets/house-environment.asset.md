# Cozy House Environment

## Purpose

The house is the play space — every room is a potential discovery zone, every hiding spot is
a game content element that MEC-001's hiding selection algorithm operates on. The environment
directly supports the Discovery aesthetic (rooms feel like "uncharted territory") and the
Fantasy aesthetic (player inhabits the babysitter role in a believable home).

## Emotional Intent

- **Primary emotion**: Coziness and safety — this home feels warm, lived-in, inviting
- **Energy level**: Low — calm environment, nothing threatening or urgent
- **Tone**: Warm, colorful, slightly whimsical — a children's book illustration brought to 3D
- **Reference**: Studio Ghibli interiors (Kiki's Delivery Service, Totoro). Warm wood tones, soft lighting, slightly oversized furniture that makes the player feel small. NOT sterile/modern — cluttered and cozy

## Technical Requirements

- **Total poly budget**: Max 50,000 triangles for all rooms combined (Roblox mobile target)
- **Room count**: 4-6 rooms (see Room List below)
- **Texture resolution**: 256x256 per prop, 512x512 per room (tiling textures where possible)
- **Lighting**: Per-room lighting rigs using Roblox PointLights and SpotLights. Each room has a distinct color temperature:
  - Kitchen: Warm yellow (Color3: 255, 220, 150)
  - Bedroom: Soft blue (Color3: 180, 200, 230)
  - Living room: Warm orange (Color3: 240, 190, 140)
  - Bathroom: Bright white (Color3: 230, 235, 240)
  - Nursery: Soft pink/lavender (Color3: 230, 200, 220)
- **Scale**: Doors ~7 studs tall, rooms ~30x30 studs. Slightly oversized proportions (whimsical)
- **Navigation**: All rooms connected, no locked doors, baby NavMesh-compatible floor surfaces

| Bedroom | 3-4 | 1 Easy, 1 Medium, 2 Hard | Bed (under), closet, laundry basket, dresser |

**Total**: 15-21 hiding spots (MEC-001 requires minimum 15)

## Hiding Spot Specification

Each hiding spot is a Part or Model tagged with CollectionService:

- **Tag**: `HidingSpot`
- **Attributes**:
  - `Difficulty` (StringValue): "Easy" | "Medium" | "Hard"
  - `Room` (StringValue): Room name for visit tracking
  - `PeekAnimation` (StringValue): Which baby peek animation works here
  - `RevealAnimation` (StringValue): Which reveal animation works here
  - `BabyOffset` (Vector3Value): Local position where baby sits when hiding
  - `BabyFacing` (Vector3Value): Direction baby faces when hiding

**Difficulty definitions** (from the player's perspective):
- **Easy**: Baby is partially visible (feet sticking out, shadow visible). Player spots them within ~5 seconds of entering the room
- **Medium**: Baby is hidden behind/inside furniture. Player must move around the room to find them. ~10-20 seconds
- **Hard**: Baby is fully concealed (inside closet, under bed). Player needs hints or thorough exploration. ~20-35 seconds

## Variants

| Variant | Trigger | Difference from base |
|---------|---------|---------------------|
| Daytime | Session start | Bright lighting, windows show sunlit exterior |
| Evening | sleepiness 3+ | Lights dim 20%, windows show sunset colors |
| Bedtime | sleepiness at threshold | Lights dim 50%, warm glow only, nightlights on |

## Roblox Integration

- **Location**: `Workspace.World.Rooms` (Folder containing room Models)
- **Instance type**: Model per room, with child Parts for walls/floor/props
- **CollectionService tags**:
  - `Room` on each room model
  - `HidingSpot` on each hiding position
  - `InteractiveProp` on props that wiggle during hints (MEC-001 Rule 4)
- **Attributes on Room models**:
  - `RoomName` (StringValue): Matches Room column in table above
  - `Visited` (BoolValue): Set by MEC-001 when player enters (for hiding spot weighting)

## Placeholder Protocol

- **Placeholder description**: Simple box rooms (6 Part walls + floor) with colored neon material matching each room's color temperature. Hiding spots are small transparent Parts at correct positions with all required attributes set. No props — just labeled anchors
- **Placeholder location**: Generated by a setup script or built manually in Studio
- **Behavioral equivalence**:
  - Room dimensions must match spec (~30x30 studs)
  - All hiding spots must exist at correct positions with correct attributes
  - NavMesh must work (flat floor, openings between rooms)
  - CollectionService tags must be applied
  - Lighting must be present (even as single PointLight per room)
- **Swap protocol**: Replace room Models in `Workspace.World.Rooms`. Code references rooms via `Room` tag and `RoomName` attribute, and hiding spots via `HidingSpot` tag with attributes. No hard-coded paths to specific props
